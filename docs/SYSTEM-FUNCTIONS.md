# Codex Float for Windows 系统功能说明

版本：v1.0.1  
平台：Windows 10 / Windows 11 x64

## 1. 产品定位

Codex Float for Windows 是 Codex 用户的桌面额度状态工具。它的核心任务是在不打开完整应用的情况下，让用户在一秒内了解本周剩余额度，并在需要时查看重置时间、套餐和重置机会。

它不负责购买、增加、绕过、预测或重置额度。

## 2. 系统组成

系统由四个部分构成：

1. **Codex 可执行文件发现器**：定位本机可用的 `codex.exe`。
2. **App-server 协议客户端**：启动本机 Codex 子进程并读取额度。
3. **额度映射层**：选择 Codex 限额桶、识别每周窗口并转换剩余百分比。
4. **Windows 桌面界面**：提供悬浮胶囊、详情面板和系统托盘。

## 3. Codex 发现与连接

### 3.1 查找顺序

- 用户通过 `CODEX_FLOAT_CODEX_PATH` 指定的路径
- Codex Desktop 的版本化安装目录
- Windows `PATH` 中的 `codex.exe`

Codex Desktop 目录会按文件更新时间选择最新版本，因此应用升级后通常无需重新配置。

### 3.2 Codex 数据目录

- 优先继承有效的 `CODEX_HOME`
- 未指定时，使用当前用户 `%USERPROFILE%\.codex`
- 仅当其中存在 Codex 登录状态时显式传递给子进程

应用本身不读取登录令牌内容。

## 4. App-server 协议

应用使用标准输入输出连接：

```text
codex app-server --stdio
```

连接流程：

1. 发送 `initialize`
2. 等待初始化响应
3. 发送 `initialized`
4. 请求 `account/rateLimits/read`
5. 解析响应并关闭本次子进程

单次请求默认等待 12 秒。超时、进程退出、无效 JSON 和服务端错误都会转换为用户可理解的状态。

## 5. 额度计算

- 优先选择 `rateLimitsByLimitId.codex`
- 回退到兼容字段 `rateLimits`
- 仅把 10,080 分钟窗口识别为每周窗口
- 剩余百分比为 `100 - usedPercent`
- 最终结果限制在 0–100 之间
- 不把短周期窗口错误标记为周额度

重置机会使用 `rateLimitResetCredits.availableCount`，仅接受 0–100 范围内的值。

## 6. 悬浮胶囊

### 6.1 视觉

- 154 × 52 像素基础尺寸
- 逐像素 Alpha 分层窗口，支持平滑圆角
- 百分比使用 26px Bahnschrift 粗体
- 背景按剩余额度比例填充
- 使用图标、数字和状态点共同表达状态，不只依赖颜色

### 6.2 颜色规则

- 剩余 > 50%：绿色
- 剩余 > 20% 且 ≤ 50%：橙色
- 剩余 ≤ 20%：红色
- 加载或未知：中性靛灰色

### 6.3 交互

- 单击胶囊：展开详情
- 单击展开面板任意位置：回缩成胶囊
- 拖动：移动窗口，不触发展开或回缩
- 始终置顶且不显示任务栏按钮

## 7. 详情面板

详情面板展示：

- 产品与周期标识
- 大号剩余百分比
- 额度进度轨道
- 最近更新时间
- 下次重置日期和时间
- 相对重置倒计时
- 当前套餐
- 可用重置机会
- 自动刷新状态

未知字段显示“暂不可用”，不使用模拟数据。

## 8. 系统托盘

### 8.1 左键

显示悬浮窗并移动到主屏幕顶部中央，用于找回被遮挡或移出可视区域的窗口。

### 8.2 右键菜单

- 显示版本 `v1.0.0`
- 在屏幕中央显示
- 立即刷新
- 开机启动
- 退出应用

菜单使用深色表面、浅色文字和较大的点击区域。

## 9. 自动刷新

- 首次显示时立即刷新
- 默认间隔 60 秒
- 防止并发重复读取
- 后台线程执行协议请求，界面线程只负责更新状态
- 失败后保留错误说明并在下一周期自动恢复

## 10. 开机启动

开机启动使用当前用户注册表：

```text
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
```

键名为 `CodexFloatWindows`，只影响当前 Windows 用户，不需要管理员权限。

## 11. 单实例

应用使用命名 Mutex `CodexFloatWindows.SingleInstance`。重复启动时，新进程会直接退出，避免多个悬浮窗和重复额度请求。

## 12. 错误状态

系统区分并显示：

- 未找到 Codex CLI
- Codex 未登录
- 当前 Codex 版本不支持额度读取
- 请求超时
- 无权访问 Codex 数据目录
- 服务端返回错误
- 未找到每周额度窗口

错误界面会给出恢复建议，并说明系统仍会自动重试。

## 13. 隐私边界

- 不包含遥测
- 不记录协议完整响应
- 不打印或持久化访问令牌
- 不向开发者或第三方服务器发送额度数据
- 只启动用户本机已安装的 Codex CLI
- 本地只保存 Windows 开机启动选择

## 14. 构建与分发

项目使用单文件 C# 源码和 Windows 自带 .NET Framework 编译器构建，不依赖第三方包。

`app.manifest` 声明 Per-Monitor DPI 感知和 Windows 兼容性。构建产物是独立的 Windows GUI 可执行文件。
